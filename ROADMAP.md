# 异动偏离预警器开发 Roadmap

本文档把产品设计拆成可以逐步开发、测试和提交的任务。每个阶段都应保持小程序可运行，接口未完成时允许回退到明确标注的模拟数据。

## 当前状态

- [x] 产品与技术方案确认
- [x] 微信小程序 UI MVP
- [x] GitHub 仓库与开发者工具配置
- [x] 后端第 1 阶段：FastAPI 接口骨架与小程序联调
- [x] 后端第 2 阶段：数据库、缓存与持久化代码
- [x] 后端第 3 阶段（候选池版）：免费行情适配器与标准化
- [ ] 后端第 4 阶段：异动规则计算已接入，待全市场覆盖与历史回放
- [ ] 后端第 5 阶段：微信登录、白名单与消息提醒
- [ ] 部署、监控和内部试运行

## 阶段 1：可运行接口骨架

目标：前端不再直接依赖页面内模拟数据，而是通过稳定的 `/api/v1` 契约访问后端。

- [x] 创建 FastAPI 模块化工程
- [x] `GET /health`
- [x] `GET /api/v1/market/status`
- [x] `GET /api/v1/anomalies/intraday`
- [x] `GET /api/v1/anomalies/confirmed`
- [x] `GET /api/v1/anomalies/history`
- [x] `GET /api/v1/securities/{symbol}`
- [x] `GET /api/v1/securities/{symbol}/deviation`
- [x] `GET/POST/DELETE /api/v1/watchlist`
- [x] `PUT /api/v1/alerts/{symbol}`
- [x] 小程序请求封装、数据适配和失败回退
- [x] API 自动化测试

验收：启动后端后，小程序首页、详情、自选和提醒状态均显示后端 Mock 数据；关闭后端时页面明确提示连接失败，不再使用页面内置股票数据。

## 阶段 2：数据库与缓存

目标：替换内存仓储，保证自选、行情和异动事件重启后不丢失。

- [x] Docker Compose：PostgreSQL 16、Redis 7
- [x] SQLAlchemy 2 数据模型
- [x] Alembic 初始化及首个迁移
- [x] `security_master`、`daily_quote`、`index_daily_quote`
- [x] `rule_version`、`anomaly_event`、`watchlist`、`alert_setting`
- [x] Repository 接口与 PostgreSQL/SQLite 实现
- [x] Redis 榜单、市场状态及过期策略
- [x] SQLite 持久化与内存缓存自动化测试
- [ ] 安装 Docker 后执行 PostgreSQL、Redis 容器级验收

验收：服务重启后自选仍存在；榜单可从 Redis 快速读取；缓存丢失后能从数据库恢复。

当前开发机未安装 Docker，因此已完成 SQLite 重启持久化、缓存失效和迁移升降级验证；生产 PostgreSQL/Redis 代码与 Compose 配置已完成，容器级验收待安装 Docker 后执行。

## 阶段 3：行情数据接入

目标：以可替换的适配器接入零付费数据源，仅用于内部验证。

- [x] AKShare/东方财富与新浪财经公开接口调研
- [x] 候选股票主数据和板块映射
- [x] 上证A指、深证A指、创业板综、科创50指数映射
- [x] 实时快照和 45 日日线字段标准化
- [x] 超时、重试、异常状态与禁用 Mock 回退
- [x] 应用内交易时段后台任务（盘中 15 秒、盘后每日一次）
- [ ] 原始响应样本和接口变更监控

验收：交易时段内 10～15 秒刷新；单个数据源失败时页面明确展示数据健康状态。

## 阶段 4：异动规则引擎

目标：按生效日期和板块运行版本化规则，计算结果可解释、可回放。

- [x] 基于真实日线的有效交易日窗口
- [x] 3 日主板 20%、创业板/科创板 30% 正向偏离值
- [x] 10 日 100%、30 日 200% 严重异常波动
- [ ] 10 日内同向异动次数规则
- [x] “单日涨停即可触发”的 3 日接近异动预测
- [ ] Decimal 精度、阈值边界和幂等事件
- [ ] 不少于 30 个历史案例回放

验收：规则单元测试覆盖边界值、停牌、窗口不足、指数缺失和重复计算。

## 每次提交约定

1. 一个提交只完成一个可验收目标。
2. 新接口必须有响应模型和自动化测试。
3. AppSecret、SMTP 授权码和数据库密码只写入 `.env`，不得提交。
4. 计算状态统一标注“系统计算”，不使用“交易所确认”。
5. 合并前执行后端测试和前端 JavaScript/JSON 静态检查。
